(D-14/D-12) to the `Stats` variant and thread them through `main.rs`; restructure

output. It consumes the reader from 08-01 and the helpers from 08-02 (hence Wave 2).

header assertions. No migration, no field/view rename, no write-path change (D-01).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args (`--since`/`--project`/`--bytes`/`--all`) → execute | User-supplied flags drive query selection and render mode. |
| stored `project_path` → canonical key → printed verbatim to stdout | A stored path string is rendered into the terminal output. |
| `.git` filesystem (via 08-02 helpers) → project rollup | Read-only `.git` resolution feeds the canonical key (mitigated in 08-02). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Tampering (SQL injection) | headline `filtered_overall_totals` call site | mitigate | The reader binds `--since`/`--project` via `?N` placeholders (08-01); execute passes them as typed `Option<i64>`/`Option<&str>`, never interpolated. Reuses the existing section binding posture. Severity → low. |
| T-08-03 | Denial of Service | headline read on a filter matching nothing | mitigate | 08-01's `COALESCE(SUM,0)` returns a zeroed headline; execute maps any reader Err→Ok(1) (no `?`→anyhow leak). No panic on empty/filtered-empty. Severity → low. |
| T-08-07 | Information Disclosure (terminal control bytes) | canonical key / project path printed verbatim | accept | A stored `project_path` could in principle contain ESC/control bytes that reach the terminal when printed. Risk is LOW (paths come from the local single-user write path's own `current_dir()`, not an external attacker) and the pre-change code already printed `project_path` verbatim — this phase does not regress it. Unlike `explain` (which sanitizes captured tool OUTPUT), `stats` prints only numeric aggregates + path keys. Documented as accepted; control-byte neutralization deferred (08-RESEARCH.md Security Domain). |
| T-08-08 | Denial of Service (unbounded output) | uncapped sections at large history sizes | mitigate | Default top-N=10 cap + "… M more" hint bounds output regardless of history size (D-11); `--all` is an explicit user opt-in to uncapped output. This is the core readability fix. Severity → low. |

No threat in this plan rises above `low` after mitigation except T-08-07, which is
explicitly ACCEPTED (low risk, no regression vs. prior behavior, local single-user
trust model). The write hot path is untouched; all new logic runs only in the `stats`
read-only cold path.
