```rust
/// WR-03: normalize a `--project` argument ... We deliberately do NOT
/// `canonicalize` (resolve symlinks): the write side stores the *logical* cwd,
/// so symlink resolution here would diverge from it.
fn normalize_project(p: &std::path::Path) -> String {
    let abs = std::path::absolute(p).unwrap_or_else(|_| p.to_path_buf());
    let s = abs.to_string_lossy();
    let trimmed = s
        .strip_suffix(std::path::MAIN_SEPARATOR)
        .filter(|t| !t.is_empty())
        .unwrap_or(&s);
    trimmed.to_string()
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored `project_path` string → `.git` filesystem resolution | A stored path (and the `.git` files it points at) drives bounded read-only filesystem access during `stats` (a cold path). |
| `$TMPDIR` env var → ephemeral prefix set | An environment value is added to the runtime prefix list. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering / DoS | `.git` ancestor walk + gitdir/commondir following | mitigate | Bounded by `path.ancestors()` (finite) and a single gitdir-hop + single commondir-hop (D-09) — no unbounded gitdir→commondir chain following. Only `fs::metadata`/`fs::read_to_string`, never a `git` subprocess (D-09), never `canonicalize()` on possibly-deleted paths (D-08). Any IO error / missing / malformed / bare / non-existent → literal fallback, never a panic (D-10). Asserted by Task 2 literal-fallback tests. Severity without mitigation: medium (path traversal / resource exhaustion); mitigated → low. |
| T-08-05 | Elevation of Privilege (write-path leakage) | canonicalization reachable from `lacon run` | mitigate | All helpers are private `fn`s in `commands/stats.rs` (D-04), unreachable from `lacon-core` and `commands/run.rs`. The `repo_root`-on-write idea is explicitly DEFERRED. Asserted by the grep gate in Task 2 acceptance criteria (no references from run.rs / lacon-core). This protects the ≤10ms cold-start write hot path (ADR 0001/0013). |
| T-08-06 | Spoofing (false ephemeral collapse) | `is_ephemeral` prefix match | mitigate | Component-wise `Path::starts_with` (D-08), not `str::starts_with`, so `/tmpfoo` is not collapsed into a real-project bucket. Asserted by the mandatory `/tmpfoo` negative in Task 1. Severity → low. |

No threat in this plan rises above `low` after mitigation. All filesystem access is
read-only, bounded, and confined to the `stats` cold path. Note: canonical keys are
printed verbatim by 08-03; control-byte neutralization of project paths containing
ESC bytes is LOW risk and left to 08-03's discretion (flagged in 08-RESEARCH.md
Security Domain).
</threat_model>
