<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| third-party GitHub Action → CI runner | Actions execute with repo permissions; a malicious/typo-squatted action could exfiltrate or tamper. |
| CI workflow → external package registries | Any `install` step would pull untrusted code into the build (and break hermeticity). |
| `#[ignore]`d real-pnpm test → network | A real `pnpm install` reaches the npm registry (supply-chain); must never run in CI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-CI-01 | Tampering / Elevation | GitHub Action dependencies in ci.yml | mitigate | Conservative baseline uses ONLY `actions/checkout@v4` plus the runner's pre-installed Rust (RESEARCH A1). Any third-party action (dtolnay/rust-toolchain, Swatinem/rust-cache) must have its slug verified and be pinned to a major-version tag or full SHA before use. |
| T-06-CI-02 | Information Disclosure | runner secrets exposure | mitigate | Phase 6 CI needs NO secrets; the workflow adds no `secrets.*`. A top-level least-privilege `permissions: contents: read` block limits the default token. |
| T-06-CI-SC | Tampering (supply chain) | npm/registry installs in CI | mitigate | The workflow runs NO `brew/npm/pip/apt install`; `rusqlite[bundled]` removes the only system-library need; the real-pnpm test is `#[ignore]`d so `cargo test --workspace` (no `--ignored`) never fetches from npm. Verified by the grep gate in Task 3. |
| T-06-CI-03 | Denial of Service (flaky gate) | hard `<10ms` wall-clock assert on shared macOS VM | accept (mitigated by design) | macOS cold-start is reported softly (min-of-N), not hard-asserted (Pitfall 1); the deterministic hard gate is the in-process `tracker_open` criterion bench. Residual risk (a noisy macOS number) is informational only and does not break the build. |

Block-on-high threshold: the only high-severity surface is supply-chain (CI actions + the real pnpm install), and both are mitigated (pinned/minimal actions; `#[ignore]`d + hermetic-by-grep). No unmitigated high-severity threats remain.
</threat_model>
