```rust
// crates/lacon-core/src/pipeline/stages.rs
use std::borrow::Cow;
use std::collections::VecDeque;
use smallvec::SmallVec;

pub type LineOut<'a> = SmallVec<[Cow<'a, str>; 2]>;

pub enum HeadTailMode { Lines(usize), Bytes(usize) }

pub enum Stage {
    StripAnsi,
    DropRegex(regex::Regex),
    KeepRegex(regex::RegexSet),
    ReplaceRegex { pattern: regex::Regex, replacement: String },
    Dedupe { last: Option<String>, max_kept: usize, repeat_count: usize, kept_so_far: usize },
    CollapseRepeated { pattern: regex::Regex, max_kept: usize, summary_template: String, kept_so_far: usize, dropped: usize },
    KeepHead { mode: HeadTailMode, lines_remaining: usize, bytes_remaining: usize },
    KeepTail { mode: HeadTailMode, ring: VecDeque<String>, byte_count: usize },
    KeepAroundMatch { pattern: regex::Regex, before: usize, after: usize, ctx_buf: VecDeque<String>, emit_after: usize },
    MaxBytes { cap: usize, written: usize, truncated: bool },
}

impl Stage {
    pub fn step<'a>(&mut self, line: Cow<'a, str>, out: &mut LineOut<'a>);
    pub fn flush<'a>(&mut self, out: &mut LineOut<'a>); // for KeepTail, CollapseRepeated, MaxBytes truncation marker
}
```

  <behavior>
    - StripAnsi: removes CSI/SGR/OSC sequences (regex `\x1b\[[0-?]*[ -/]*[@-~]` plus OSC `\x1b\][^\x07]*\x07`); leaves rest of line intact.
    - DropRegex(re): line emitted IFF !re.is_match(line).
    - KeepRegex(set): line emitted IFF set.is_match(line). Empty set (no patterns) → drop everything.
    - ReplaceRegex { pattern, replacement }: emits `pattern.replace_all(line, replacement)` once per line.
    - Dedupe { max_kept, .. }: collapses consecutive identical lines; emits up to `max_kept` (default 1) before suppressing duplicates; resets state on a different line.
    - CollapseRepeated { pattern, max_kept, summary_template }: while consecutive lines match `pattern`, emit up to `max_kept` examples, then suppress; when run ends OR a non-matching line arrives, emit a summary line with `{count}` substituted by total dropped (incl. examples).
    - KeepHead { Lines(n) }: emits first n lines, drops rest. KeepHead { Bytes(n) }: emits lines until cumulative emitted bytes (incl. trailing `\n`) > n; final line is truncated to fit.
    - KeepTail { Lines(n) }: rolling ring of last n lines, emitted at flush. KeepTail { Bytes(n) }: ring tracks byte count, pop-front when total > n.
    - KeepAroundMatch { pattern, before, after }: standard grep -B/-A semantics; on match emit `before` preceding context (held in `ctx_buf` ring of size `before`) + the match + next `after` lines.
    - MaxBytes { cap }: track bytes written; once `written + line.len() + 1 > cap`, set `truncated = true`, emit a final line `[lacon: truncated, <delta> more bytes dropped]` (byte-exact per D-08), drop subsequent lines.

    Each primitive variant has a `step(line, &mut out)` and (where stateful) a `flush(&mut out)` that runs once at end-of-stream.

Critical implementation notes (per RESEARCH.md):

1. **`KeepRegex(RegexSet)` is the OR-merged variant.** A `Pipeline::new` builder turns N adjacent `Stage::KeepRegex(set)` entries into one `Stage::KeepRegex(RegexSet::new(all_patterns))`. Per D-06 + Pitfall 5: do NOT use `RegexSet` for `drop_regex` or `replace_regex` — those need `Match` / `Captures`, which `RegexSet` cannot provide. Each `DropRegex` and `ReplaceRegex` stage carries its own `regex::Regex`.

2. **`MaxBytes` truncation marker is byte-exact.** Format string: `"[lacon: truncated, {} more bytes dropped]"` (note: NO leading newline, NO trailing newline appended by the marker itself — the pipeline runner adds the trailing `\n` when joining the output). The `{}` is replaced by the count of bytes that would have been emitted but were dropped (including the rest of the current line + every subsequent line + their `\n` separators).

3. **`StripAnsi` regex**: prefer a single compiled `regex::Regex` covering CSI (`\x1b\[[0-?]*[ -/]*[@-~]`) + OSC (`\x1b\][^\x07]*(\x07|\x1b\\)`) sequences. Compile once in a `OnceLock<regex::Regex>` (Rust 1.70+ stable, no `lazy_static!` per RESEARCH State of the Art) so all `StripAnsi` stages share the regex. Acceptable to inline the pattern; no need for the `strip_ansi_escapes` crate (RESEARCH "Don't Hand-Roll" lists it as optional).

4. **`KeepTail` bytes mode** (RESEARCH Pattern 3 callout): the `VecDeque<String>` ring must track a running `byte_count`; on each push, if `byte_count + line.len() + 1 > cap_bytes`, pop_front lines until `byte_count` fits. At `flush`, drain the ring into `out`.

5. **`KeepAroundMatch`**: the `ctx_buf: VecDeque<String>` is the rolling "before" window of size `before`. On match: drain `ctx_buf` to `out`, emit the matching line, set `emit_after = after`. On non-match while `emit_after > 0`: emit, decrement; otherwise push to `ctx_buf` (popping front if size > `before`).

6. **`Pipeline::new`** constructor logic:
   - Walk `stages` left-to-right.
   - Collect adjacent `Stage::KeepRegex(set)` into a single `RegexSet` (concat all patterns from each set's `.patterns()` into a `Vec<&str>`, then `RegexSet::new` on the merged list).
   - Note: actual schema parsing (PLAN-03) emits one `Stage::KeepRegex(RegexSet::new([single_pattern]))` per `keep_regex:` YAML stage; the constructor merges N of those into one. Document this contract in a comment.

7. **`Pipeline::run<I: Iterator<Item = String>>(&mut self, lines: I) -> Vec<String>`**: For each input line, call `stage[0].step(line, &mut tmp1)`; for each emission in `tmp1`, call `stage[1].step(emission, &mut tmp2)`; etc. End with `flush()` propagation across stages. Reuse two `LineOut` SmallVecs swapped between stages to avoid allocation per line.

8. **`OnceLock` for the StripAnsi regex** — single shared compile across all StripAnsi stages.

Add unit tests inline in `stages.rs` (one `#[cfg(test)] mod tests` block) that probe edge cases the golden fixtures don't cover (e.g., RegexSet OR-merge of 3 keep_regex stages → single match call; max_bytes overflow exactly at boundary; keep_around_match where matches overlap).

```rust
//! Golden-fixture tests for the 10 native primitives.
//!
//! Per VALIDATION.md "Per-Primitive Unit Test Pattern":
//! - Each primitive has tests/fixtures/primitives/<name>/input.txt + expected.txt
//! - Test reads input, builds a single-stage Pipeline, asserts byte-exact match against expected
//!
//! All fixtures live at the WORKSPACE root (`tests/fixtures/...`), not the
//! crate root, because they are shared with PLAN-05/PLAN-07 integration tests.

use lacon_core::pipeline::{Pipeline};
use lacon_core::pipeline::stages::{Stage, HeadTailMode};
use regex::{Regex, RegexSet};
use std::path::PathBuf;

fn fixture_path(primitive: &str, name: &str) -> PathBuf {
    // Tests run from the workspace root by default.
    PathBuf::from("tests/fixtures/primitives").join(primitive).join(name)
}

fn run_fixture(primitive: &str, mut stages: Vec<Stage>) -> (String, String) {
    let input_path = fixture_path(primitive, "input.txt");
    let expected_path = fixture_path(primitive, "expected.txt");
    let input = std::fs::read_to_string(&input_path)
        .unwrap_or_else(|e| panic!("read {}: {}", input_path.display(), e));
    let expected = std::fs::read_to_string(&expected_path)
        .unwrap_or_else(|e| panic!("read {}: {}", expected_path.display(), e));

    let mut pipeline = Pipeline::new(stages);
    let lines: Vec<String> = input.lines().map(str::to_owned).collect();
    let out = pipeline.run(lines.into_iter());
    let actual = out.join("\n");

    // Trim trailing newline from expected (text editors add one); compare normalised.
    let expected_trimmed = expected.trim_end_matches('\n').to_owned();
    (actual, expected_trimmed)
}
```
