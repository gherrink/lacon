| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-10 | DoS (local) | malformed settings.json / config.yaml / rule YAML crashing doctor | mitigate | All parsing goes through serde_json/validate_file/load_all which return Result; doctor maps every error to a printed line + non-zero exit, never a raw `?` panic (Task 1 acceptance; init.rs NotFound-as-normal precedent) |
| T-04-11 | Tampering | doctor writing to history.db during the health check | mitigate | Check 5 uses `open_readonly` (D-08); grep gate asserts `Tracker::open` is never called from doctor.rs |
| T-04-12 | Spoofing | a stale/forged hook entry passing the fingerprint check | accept | Doctor reports presence of the `lacon-claude-hook` fingerprint, matching init's write contract (A4); it is a local self-check, not a security boundary — a user editing their own settings.json is in the trust domain |
| T-04-13 | Information disclosure | doctor printing config/rule contents | mitigate | Doctor prints only the offending PATH + the validator's structured error message, not raw file contents/secrets |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: zero installs) |
