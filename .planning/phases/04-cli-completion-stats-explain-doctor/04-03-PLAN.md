<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args (`--project`/`--since`/`--rule`/`id`) → query layer | Untrusted user strings reach the SQL parameters and the id parser |
| stored raw output → terminal render | Previously-captured bytes (which may contain ANSI/control sequences) are printed in the explain side-by-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | `--project`/`--rule` reaching SQL | mitigate | Values pass through to `tracking::query` which binds them as `?N` params (plan 01); stats.rs never builds SQL — D-01 keeps it in core (grep gate: no `rusqlite` in stats.rs) |
| T-04-07 | DoS (local) | `lacon explain abc` panic on parse | mitigate | `id.parse::<i64>()` matched, never `unwrap()`; bad input → clean message + non-zero exit (Task 2 acceptance + cli_explain test) |
| T-04-08 | Information disclosure | explain shows stored raw output | accept | Raw output is off-by-default (ADR-0009); explain shows only what the user opted to store, errors (not fabricates) when retention was off; raw pruned at 3 days |
| T-04-09 | Tampering (terminal injection) | ANSI/control sequences in stored raw output rendered to the terminal | mitigate | The side-by-side render treats stored output as data printed on the LEFT (raw) column; the filtered column reflects pipeline output (rules typically include `strip_ansi`). Render line-by-line via normal `println!` of owned Strings (no raw fd passthrough); the user explicitly invoked `explain` on their own stored data. Document that raw column reproduces stored bytes verbatim for fidelity (Phase 6 SC3) — escaping is NOT applied to the raw column because byte-fidelity is the contract; the filtered column is the safe-to-read view |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | Only `insta` (already a workspace dev-dep, real/widely-used) may be added as a dev-dependency; no runtime install; RESEARCH audit: zero new packages |
</threat_model>
