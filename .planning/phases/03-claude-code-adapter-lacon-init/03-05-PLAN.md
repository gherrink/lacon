<!-- Target shape of .claude/settings.json after `lacon init`. -->

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          { "type": "command", "command": "lacon-claude-hook" }
        ]
      }
    ]
  }
}
```

Idempotency contract (D-12, D-28):
- Re-running `lacon init` walks `hooks.PreToolUse[]`. For each matcher-group with `matcher == "Bash"`:
  - Filter out inner `hooks[]` entries where `command` field `.starts_with("lacon-claude-hook")`.
- Drop now-empty Bash matcher groups.
- Append the canonical lacon entry.

Preservation contract (D-28):
- Non-Bash matcher groups (e.g. `matcher: "Edit"`) — untouched.
- Bash matcher groups with non-lacon hooks (e.g. user's formatter) — non-lacon hooks preserved; only lacon hooks scrubbed-and-reinserted.
- Top-level keys outside `hooks` (e.g. `model`, `theme`) — untouched.

CLAUDE.md block (D-14) — example body (planner's exact wording is discretion, MUST include `!!` and `LACON_DISABLE=1` keywords):
```markdown
<!-- lacon:start -->
Bash output is filtered by lacon to reduce token usage. Bypass one command
with `!!` prefix (e.g., `!! pnpm test`). Disable filtering entirely with
`LACON_DISABLE=1`.
<!-- lacon:end -->
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User's pre-existing `.claude/settings.json` → init parse+write | The file may contain arbitrary user config (model, theme, non-lacon hooks). Init must NEVER drop or corrupt non-lacon keys. |
| User's pre-existing `CLAUDE.md` → init read+write | The file is human-curated; init must NEVER damage non-lacon content. |
| Concurrent `claude` startup ↔ `lacon init` running | Atomic write via tempfile + persist prevents partial-write corruption. |
