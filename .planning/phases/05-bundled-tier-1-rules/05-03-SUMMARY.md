## Decisions Made
- **NO rewrite (D-11):** The plan and RESEARCH locked this — `--silent` is the only cross-manager flag, but it suppresses ALL error output on failure for npm and pnpm (0 bytes, exit 1), so the E404 error is destroyed before lacon sees it. Pipeline-side filtering is strictly better and is the security control here (threat T-5-04 mitigated).
- **Extra yarn-1 noise drops:** Added `^warning ` and `^success ` drops (yarn 1.22 emits `warning package.json: ...` / `success Saved lockfile.`) on top of the RESEARCH drop list, since yarn 1 is the Corepack default on this machine. These do not touch the kept summary lines (`added`, `vulnerabilities`, `Done in`).
