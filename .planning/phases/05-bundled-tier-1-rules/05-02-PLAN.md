Rule field layout (RuleFile, schema.rs:27-64; deny_unknown_fields — any typo fails load):
`id` (kebab-case, must equal resolve id) · `description` · `match` (renamed match_spec) ·
`extends` (optional, may carry `bundled/` prefix) · `pipeline` (Vec<StageSpec>) ·
`on_error.pipeline` (own Vec<StageSpec>) · `rewrite` (omit here).

Stage YAML keys (StageSpec, schema.rs:177-212, snake_case) — exact forms:
- `- strip_ansi`  (bare unit)
- `- drop_regex: '<re>'`
- `- keep_regex: '<re>'`  (ADJACENT keep_regex OR-merge; non-adjacent AND — Pitfall 1)
- `- keep_around_match: { pattern: '<re>', before: N, after: N }`  (BOTH before AND after REQUIRED — no defaults, deny_unknown_fields, PATTERNS delta #2)
- `- keep_tail: { lines: N }`
- `- dedupe: { max_kept: 1 }`
- DO NOT hand-place `max_bytes` — loader auto-injects 32768 on BOTH success and on_error (D-07).

extends semantics (merge_rules, loader.rs:555-571, ADR-0012): parent pipeline is
PREPENDED to child's; scalar fields (on_error, match, description) inherited when
child omits them. `extends: bundled/test-base` and `extends: test-base` both resolve.

regex crate is RE2 — NO look-around `(?=)/(?!)/(?<=)`, NO backreferences `\1` (Pitfall 3).
Use keep_regex whitelist instead of negated drop_regex.
