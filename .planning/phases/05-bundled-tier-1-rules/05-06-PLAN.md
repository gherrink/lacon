<objective>
Author the `docker-build` rule (docker build / docker buildx build) and its two
fixtures. The reduction target is BuildKit progress noise: CACHED layer hits,
DONE timing lines, and `sha256:` byte-progress lines (which vary run-to-run —
Pitfall 6, so they MUST be dropped for a deterministic fixture). Step headers and
RUN command echo output are preserved; on failure the framed ERROR block and the
Dockerfile excerpt are preserved.

Purpose: Prove a rule against streaming build output with nondeterministic progress
lines, and that dropping those lines yields a deterministic, byte-stable expected.txt.
Output: docker-build.yaml, 2 fixtures from real BuildKit output.
</objective>
